Firefox Contacts: Upgraded to version 0.4

Firefox Contacts has been updated to version 0.4

Click here to open the Contacts management screen.

What's new in this release?

Support for Groups

Contacts now supports the collection of contacts into named groups. Groups are displayed in the contact list. When a web page asks for access to your contacts, you may now restrict access to one, or some, of your groups. A contact can belong to any number of groups.

The groups that Contacts displays are imported from your contact sources; if you have groups set up in your Google contact list, or your native address book application, they will be imported. Firefox Contacts will also automatically create a group for each of your contact services.

If you use exactly the same name for a group on two or more services, the groups will be combined.

Support for OAuth logins

Where possible, Contacts now uses the industry-standard OAuth login mechanism to access websites. You will need to re-connect your browser to your services once to set it up.

Support for merge and split operations

The Contact manager now allows you to merge records from different services that represent the same person, or to split records that were automatically merged.

Last.fm integration

Contacts can import friend lists from Last.fm.

"Export" command

Contacts can export the unified list of contacts as a VCard file for use in other applications.

Javascript API documentation

Documentation for the Contacts API is now posted on our site. Web developers are encouraged to experiment with it and provide feedback.

Experimental "services" API

An opt-in service integration API is included as an experiment. Developers can read about it and how to enable it for experimental hacking here.

Please visit our site if you have any comments or feedback.


Release Notes:

  • Visual refresh with new styling.
  • Support for Groups. The Portable Contacts "tags" attribute is used to construct a list of groups; each contact may belong to as many groups as desired. Each importer automatically adds its own name as a tag.
  • Support for OAuth login for many contact services through the OAuthorizer library. This also involved changing the GMail importer from VCard to the Google Contacts API.
  • Reworked web content permission dialog. The user is prompted to select groups of contacts to make available to web content.
  • When permission is granted to a website, the permission is saved until Firefox leaves that page. If the "Remember" checkbox is checked, it is saved across multiple runs of Firefox.
  • Reworked internal database format to support merge and split operations. Merge and split should correctly persist across refreshes of a service.
  • Improved performance of contact discovery services.
  • Support for postal address and group import from Mac address book.
  • Support for "callto:" protocol for phone numbers.
  • Added "export" function to manager UI
  • Added HCard markup to person detail pages
  • Changed Javascript API to match current W3C Contacts draft.
